### Cambios en DB (`schema.sql` y `schema.dbml`)

**Nuevos enums:**
```sql
CREATE TYPE subscription_status    AS ENUM ('trialing', 'active', 'past_due', 'cancelled', 'expired');
CREATE TYPE payment_order_status   AS ENUM ('pending', 'paid', 'rejected', 'cancelled', 'expired');
CREATE TYPE payment_provider       AS ENUM ('flow');
```

**Nuevas tablas:**
```
subscription_plans  — catálogo de planes; PK uuid; campos: slug (unique), price, billing_interval_days, features (jsonb), is_active
subscriptions       — suscripción activa; user_id UNIQUE (1:1); campos: plan_id, status, current_period_start, current_period_end, cancelled_at
payment_orders      — orden de pago; campos: commerce_order (unique), flow_token (unique), flow_order, provider_response (jsonb), status
```

**FKs nuevas:**
```
subscriptions.user_id       → users.id
subscriptions.plan_id       → subscription_plans.id
payment_orders.user_id      → users.id
payment_orders.subscription_id → subscriptions.id
payment_orders.plan_id      → subscription_plans.id
```

### Endpoints habilitados

| Método | Path | Auth | Descripción |
|--------|------|------|-------------|
| GET | `/subscriptions/plans` | No | Lista planes activos |
| GET | `/subscriptions/me` | JWT | Suscripción del usuario autenticado |
| POST | `/subscriptions/checkout` | JWT | Inicia pago → devuelve `{ paymentUrl }` |
| POST | `/subscriptions/webhook` | No | Callback Flow (firma verificada) |
| GET | `/subscriptions/return` | No | Página HTML resultado post-pago |
| POST | `/subscriptions/return` | No | Ídem, forma alternativa que usa Flow |
